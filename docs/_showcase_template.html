How it works

Don't analyse the wave. Let the ear hear it.

The same four stages run in the browser: your wave is coupled into an audio-rate drive, a biomimetic cochlea resolves it, its features are read off the neurogram, and a fixed bank of scale notes resynthesises it — so the result is always in-tune and unmistakably its category.

01 · TRANSDUCTION

Physical coupling

Each wave becomes an audio-rate drive; gamma peaks become photon pings.

02 · THE EAR

Cochlea + hair cells

An ERB gammatone filterbank and hair-cell rectify / compress / adapt.

03 · PERCEPTION

Auditory features

Brightness, loudness, onset rate and tonality — from the neurogram, not an FFT.

04 · MUSIC

Note-bank synthesis

A fixed bank of scale notes, plus reverb, vibrato and tremolo.

Six voices

Every category keeps its own sound.

Comparable waves sound almost identical; very different waves spread apart — yet each category holds a fixed timbral fingerprint. These are rendered by the Python package; press play.

Interpretability

Hear the difference: the ear vs a plain FFT.

The naive approach — turn the loudest spectral bins into tones — is inharmonic and category-agnostic. The same wave through the ear is scale-locked and keeps its identity. Play both, then read the numbers.

Quantitative

Read the wave back off the sound.

Because the mapping is continuous and monotonic, physical parameters are recoverable by ear.

Under the hood

The code, two ways.

Wav2Aud is a Python package (NumPy / SciPy) with a full offline pipeline, a stateful real-time engine and ROS 2 nodes. This page ports the core to the browser so you can try it with no install.

Python package

  • transduction.py — six physical couplings (heterodyne, Doppler, AM/FM demod, thermal, photon, seismic).
  • cochlea.py / haircell.py / ear.py — gammatone filterbank, hair cells, bat fovea, antenna resonance.
  • mapping.py / synthesis.py — category voices + phase-continuous note-bank synthesis.
  • realtime.py — stateful streaming engine (complex gammatone, Schroeder reverb).
  • metrics.py / geometry.py — wave↔audio measures + birdsong-style geometry.
  • ingest.py / ros/ — bring-your-own-waveform + ROS 2 perception nodes.
import wav2aud as w2a
w2a.sonify(w2a.simulate.preset("radar_car")).write("out.wav")

This page (browser port)

  • The analysis — coupling, a 2×biquad gammatone cochlea, hair-cell envelopes and features — runs in plain JavaScript on your file.
  • The synthesis — the fixed scale-note bank with the category's partials, vibrato, tremolo and reverb — is rendered by the browser's Web Audio engine (an OfflineAudioContext).
  • The 3-D scene is a delay-embedding attractor of your wave, drawn on a canvas and driven by an analyser as it plays.
  • The scale slider re-sonifies your wave across a typical→extreme sweep to map the auditory range.

It's a faithful, compact port — the Python package remains the reference implementation for research and robotics.

Live & in the field

A real-time sense for robots.

The stateful engine keeps the cochlea, hair cells and oscillator phases continuous across blocks, so a live sensor feed streams to seamless audio. A hardware-agnostic WaveSource feeds it from a file, an array or a driver.

Where it's useful

  • Eyes-free monitoring — hear a Geiger stream or seismometer without a screen.
  • Robotics feedback — sonar / radar returns as audio an operator or audio-ML model can read.
  • Field triage — audition hours of seismic or radio data; anomalies jump out.
  • Outreach — an honest, note-by-note answer to “what does gamma sound like?”
